在MC的游玩中，AA(All Advancements,全成就)是一种十分有观赏性的玩法，在观看了许多MC的速通视频后，我也开始尝试着去做，虽然耗时20.5个小时(也是当半个生存档玩了)，经历了一些挫折，甚至有不少成就都是一边学一边做的，不过在好运的加持下，也是顺利地完成了第一次全成就。
![胜利结算画面](/blog/essays/my_first_MCAA/myfirstAA.png)
## 开场
首先我是接着自己的RSG PB(Random Seed Glitchless Personal Best,随机种子速通个人最佳成绩)玩的，学着练了一个月，虽然一循环炸龙和速搭一点都不会，不过也是来到了sub38(即低于38分钟，由于用的专门的速通整合包，计时器就一直这么挂着没动)，进折跃门看了下后发现末地船不远，也就抱着试一试的心态开始了AA，由于担心自己做什么任务时暴毙，探末地城一直到拿了十个鞘翅搞到了全套顶配才走。
![RSG PB](/blog/essays/my_first_MCAA/sub38+firstAA.png)
## 经过
说实话，即使完成了一遍AA我也并不清楚很多成就的具体任务，因为不追求速度而仅仅是完成，在开着[AATool](https://github.com/DarwinBaker/AATool)的情况下，基本是缺啥搞啥，不过看了这么多AA，对一些有难度的成就总是有印象在的。

跟弩相关的比较有印象的成就是一箭双雕和劲弩手，在我看的AA中这俩不算很简单的任务，因为对生物血量的控制有要求，印象中一只幻翼我用力量五的弓射了一箭，另一只给了几拳，牛羊猪鸡鱼凭感觉用斧子稿子拳头打几下就一次完成了，可能是有新手保护期吧

探索的时光我最后完成的是竹林，第一个找到的雨林没有竹林，不过有成对的豹猫，而第二个雨林有竹林和成对的熊猫。成双成对的话我最后完成的是驴，刻意去找的话总是只有一只，所以最好在早期探索的时候多关注一下。百猫全书没卡过猫，因为比较近的沼泽有女巫小屋，运气不错。

然后就全是重量级了，首先是魔女审判，我是大概在10h左右做的，当时情况相当刺激，首先是我在雪屋治疗好村民刷了本引雷出来(当时是希望他刷经验修补或引雷的，只能说幸好先出的引雷)，然后在出来在海上跑图的时候就开始下雨了，一声雷响直接给我吓的狂用火箭找叉哥，探了三四个遗迹也是非常给面子爆了，然后又狂丢火箭找村庄完成了魔女审判，完成后没过一分钟雨就停了，也是相当惊险，可以说引雷和三叉戟是要相当优先去考虑的物品了。

然后就是不稳定的同盟，印象中我用爆了3个钓鱼竿都不知道要怎么搞，而且开了一堆地狱传送门后就导致后面搞哨塔和海底神殿的时候各种串门，还是上B站严肃学习后才知道要提前放TNT和在点燃离开前拉高渲染距离的，属于是比较初见杀的成就。

至于资深怪物猎人要注意的就是袭击，一定一定一定要先把铁傀儡干掉，不然它可能会抢先把唤魔者和恼鬼干掉导致你要再打一次袭击。

![药水](/blog/essays/my_first_MCAA/potion.png)

最后就是名副其实的最难成就——为什么会变成这样呢，有相当多"有趣"的地方。第一点是炼药，我是在超平坦世界自己炼了一遍整理出来后对着做的;第二点是卡信标的双效果(急迫+跳跃)，只能说我是瞎按出来的;第三点是失明炖菜，由于在搞蜜蜂的时候顺手采了各种花花草草并且包括蓝花美耳草，所以不用刻意去找(虽然当时不知道要用上，不过第一次玩当然得谨慎点多搜集些物资);第三点是潜影贝，由于不熟练用钓竿长距离运输所以基本是慢慢运的，甚至用爆了两根钓竿才意识到末地主岛出生点好像是强加载要先远离才能用钓竿远距离运;第四点是凋零玫瑰，我是在末地卡祭坛的基岩打的凋零，但是一开始鱼没能给炸死，只能又到主世界搞了两桶鱼，如果你试过就知道这个位置的凋零在祭坛开启的情况下是很难攻击到鱼的，于是我就打算复活龙，但是水晶又被凋零打爆了……当时我都想放弃了，但是我回到主世界后在出生点发现了两条生鲑鱼和一个凋零玫瑰(大概是我我往祭坛上倒鱼的时候恰巧被凋零打死了)，也是非常有戏剧性和救赎感了。最后!就是导致我第一次尝试"为什么会变成这样呢"失败的罪魁祸首——海底神殿，我出生点附近这么大一片海楞是没有一个近的海底神殿，我去的海底神殿甚至是第二近的(右下角那个，第一近的在左上角)，当然仅仅是这样走地狱时间也绰绰有余，但是!什么叫被命名的海豚在太远的时候仍然需要呼吸而因为海豚ai的问题不会动从而导致溺死?总之我的第一次尝试因为没有吃到海豚buff而失败，不过不幸中的万幸是我接下来的第一个沙漠神殿就开出了附魔金苹果，并且在给海豚圈养起来水底铺满灵魂沙后就顺利完成了这个最难成就，如果各位打算尝试这个挑战请务必确保海豚的存活。

![海底神殿分布](/blog/essays/my_first_MCAA/monument.png)

## 尾声
结束了第一次AA的旅程也是五味杂陈啊，有很多地方惊喜万分，也包含了一点点的怅然若失，当然更多的是感动，即使一开始一无所知，也终究是一次完成，能改进的地方很多，我也会尝试进一步挑战自我，优化过程，缩短时间，不过沉淀其中的，最难忘的想必就是这次经历吧。

<script setup>
import { onMounted } from 'vue'

onMounted(() => {
  if (typeof window !== 'undefined') {
    const copyLinks = document.querySelectorAll('.inline-copy-link');
    
    copyLinks.forEach(link => {
      if (link.hasAttribute('data-bound')) return;
      link.setAttribute('data-bound', 'true');

      // 动态注入提示框的 HTML 结构，避免直接写在 MD 里被解析器误伤
      const tooltip = document.createElement('span');
      tooltip.className = 'copy-tooltip';
      tooltip.innerText = '已复制！';
      link.appendChild(tooltip);

      link.addEventListener('click', () => {
        const textToCopy = link.getAttribute('data-copy') || '';
        
        // 兼容性复制逻辑
        const performCopy = (text) => {
          if (navigator.clipboard && typeof navigator.clipboard.writeText === 'function') {
            return navigator.clipboard.writeText(text);
          }
          return new Promise((resolve, reject) => {
            try {
              const textArea = document.createElement('textarea');
              textArea.value = text;
              textArea.style.position = 'fixed';
              textArea.style.top = '-9999px';
              document.body.appendChild(textArea);
              textArea.focus();
              textArea.select();
              const successful = document.execCommand('copy');
              document.body.removeChild(textArea);
              successful ? resolve() : reject(new Error('copy failed'));
            } catch (err) {
              reject(err);
            }
          });
        };

        performCopy(textToCopy)
          .then(() => {
            // 显示提示框
            tooltip.classList.add('show');
            
            // 1.5 秒后隐藏
            setTimeout(() => {
              tooltip.classList.remove('show');
            }, 1500);
          })
          .catch(err => {
            console.error('复制失败:', err);
          });
      });
    });
  }
})
</script>
